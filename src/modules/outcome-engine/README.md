# Outcome Engine v1

## Purpose
The Outcome Engine serves as the ultimate arbiter of success for AiX OS. It does not generate advice; it measures whether the advice generated by the system (Morning Brief -> Decision Engine -> Action Center) was *correct* and *acted upon*. This acts as the quantitative feedback loop.

## Integration Lifecycle
1. The **Decision Engine** ranks an Opportunity as Priority 1 (Score 98).
2. The **Action Center** displays it as a Pending Task.
3. The user clicks **Complete** (Won) or **Ignore**.
4. The `ActionCenterService` intercepts this transition and automatically fires an `OutcomeRecord` over to the `OutcomeEngineService`.
5. The **Outcome Engine** logs it and recalculates platform-wide KPIs (`ConversionRate`, `DecisionAccuracy`, `IgnoredRate`).

## Future Learning Loop
Because all data is deterministically mapped (Decision ID → Action ID → Outcome), in future sprints this data stream will act as the direct training set for a Reinforcement Learning (RL) model.
If the engine consistently suggests properties over €400k as "Priority 1" but the `IgnoredRate` for those recommendations approaches 80%, the RL layer will organically learn to de-prioritize that vector without requiring manual rule adjustments.
